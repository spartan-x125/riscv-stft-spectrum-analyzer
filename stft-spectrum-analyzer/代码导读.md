# 代码导读

## 整体数据流

```text
dataset 音频文件
    -> audio_io 解码为按声道保存的 float PCM
    -> 每个声道独立执行 STFT 加窗分帧
    -> FFT
    -> 幅度谱与 dB 转换
    -> 梅尔滤波器组
    -> PNG / SVG / TXT 输出
```

## 文件职责

- `src/main.cpp`：扫描数据集、调用解码和 STFT、写出所有结果。
- `include/settings.h`：集中定义目标频率、数据集路径、STFT 参数和可选功能开关。
- `src/audio_io.cpp` / `include/audio_io.h`：音频解码。优先使用 FFmpeg 库；无库时 WAV 原生解码，非 WAV 尝试调用 `ffmpeg` 命令。
- `include/stft.h`：STFT 对外统一接口和 RVV 条件接口声明。
- `src/stft_scalar.cpp`：标量 STFT、标量 FFT、Hann 加窗、梅尔频谱和公共帧处理函数。
- `src/stft_adaptive.cpp` / `include/stft_adaptive.h`：根据帧长、帧数和 RVV 向量长度选择标量、帧内 RVV 或多帧 RVV。
- `src/stft_intra_rvv.cpp`：RISC-V RVV 帧内向量化，包括加窗、FFT 蝶形和幅度谱计算。
- `src/stft_multi_rvv.cpp`：RISC-V RVV 多帧转置布局、跨帧 FFT 和跨帧幅度谱计算。
- `src/image_writer_scalar.cpp`：PNG/SVG 标量输出实现。
- `src/image_writer_rvv.cpp`：RISC-V RVV 频谱图绘制加速路径。
- `src/uac_input.cpp` / `include/uac_input.h`：UAC/PortAudio 实时输入接口预留。
- `src/web_spectrogram.cpp` / `include/web_spectrogram.h`：网页实时频谱推送接口预留。

## 核心算法

STFT 流程为：

```text
x[n] -> Hann window -> frame[k] -> FFT(frame[k]) -> |X[k]| -> 20log10(|X[k]|)
```

Hann 窗用于降低频谱泄漏：

```text
w[n] = 0.5 - 0.5*cos(2*pi*n/(N-1))
```

FFT 使用 Cooley-Tukey radix-2 DIT 结构。标量版本对 `std::complex<float>` 做位倒序和蝶形运算；RVV 版本先拆分为 `real[]` 和 `imag[]`，再用 `vle32/vfmul/vfadd/vfsub/vse32` 并行处理内层蝶形。

梅尔频谱通过三角形滤波器组把线性频率 bin 映射到 Mel 频率带。当前标量路径和 RVV 路径都复用同一套梅尔滤波器逻辑，以保持输出一致。

## 调用链

```text
main()
  -> DecodeAudioFile()
  -> ComputeStft()
      -> SelectStftVectorizationMode()
      -> ComputeStftGeneric() / ComputeStftIntraRvv() / ComputeStftMultiRvv()
  -> WriteSpectrogramPng()
  -> WriteTargetFrequencySvg()
  -> WriteTargetFrequencyTxt()
```

## RVV 加速说明

RVV 加速覆盖以下环节：

- 帧内加窗：连续样本乘 Hann 窗。
- 帧内 FFT 蝶形：拆分实部/虚部后向量化复数乘加。
- 幅度谱：向量化 `sqrt(re*re + im*im)`，dB 转换保留标量 `log10`。
- 多帧路径：按同一采样位置跨帧批量加窗，转置存储后让向量 lane 对应不同帧，并行执行 FFT 蝶形和幅度谱。
- 梅尔频谱：RVV 路径使用 `vfmacc` 乘加与 `vfredusum` 归约三角滤波器覆盖的连续 bin。
- 频谱图绘制：向量归约 min/max，批量归一化 dB 值并计算 RGB 分段映射，再生成 PNG 像素。

构建时只有同时满足 `USE_RVV_STFT` 和 `__riscv_vector` 时，RVV 源码才参与有效编译。x86/ARM 平台始终走标量实现。
