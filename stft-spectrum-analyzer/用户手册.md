# 用户手册

## 1. 准备音频

把待分析音频放入仓库根目录的 `dataset/`：

```text
dataset/1.wav
dataset/music.mp3
```

WAV 可直接读取；MP3/FLAC 等格式需要 FFmpeg 库或系统 `ffmpeg` 命令。

## 2. 编译

Linux、macOS、RISC-V：

```bash
cmake -S stft-spectrum-analyzer -B stft-spectrum-analyzer/build -DCMAKE_BUILD_TYPE=Release
cmake --build stft-spectrum-analyzer/build
```

Windows：

```powershell
cmake -S stft-spectrum-analyzer -B stft-spectrum-analyzer/build
cmake --build stft-spectrum-analyzer/build --config Release
```

## 3. RISC-V RVV 编译

```bash
cmake -S stft-spectrum-analyzer -B stft-spectrum-analyzer/build -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_CXX_FLAGS="-O3 -march=rv64gcv -mabi=lp64d"
cmake --build stft-spectrum-analyzer/build
```

如果是 rv32 平台，请按芯片和 SDK 改成对应 `-march` / `-mabi`。

## 4. 运行

```bash
./stft-spectrum-analyzer/build/stft-spectrum-analyzer
```

Windows：

```powershell
.\stft-spectrum-analyzer\build\Release\stft-spectrum-analyzer.exe
```

运行时会打印平台和 RVV 状态：

```text
Platform: RISC-V / Other
RVV STFT acceleration: Enabled / Disabled
```

## 5. 查看结果

结果在：

```text
stft-spectrum-analyzer/result/
```

每个音频对应：

```text
*-spectrogram.png
*-mel_spectrogram.png
*-target_freq.svg
*-target_freq.txt
```

TXT 每行格式：

```text
时间(秒)    幅度(dB)
```

## 6. 修改目标频率和 STFT 参数

编辑：

```text
stft-spectrum-analyzer/settings.h
```

常用参数：

```cpp
#define TARGET_FREQUENCY 1000.0f
#define STFT_FRAME_SIZE  1024
#define STFT_HOP_SIZE    512
```

修改后重新编译。

## 7. UAC 实时输入

项目已提供 `IAudioInputStream` 和 `UacAudioStream` 接口。启用步骤：

1. 安装 PortAudio 开发库。
2. 将 `ENABLE_UAC_INPUT` 改为 `1`。
3. 重新 CMake 和编译。
4. 接入 USB Audio Class 设备。

当前主流程仍以文件批处理为主，UAC 接口用于后续实时音频流接入。

## 8. 网页实时频谱

项目已提供 `WebSpectrogramServer` 接口占位。启用步骤：

1. 将 `ENABLE_WEB_SPECTROGRAM` 改为 `1`。
2. 接入 WebSocket 后端库。
3. 运行程序后访问 `http://localhost:9002`。

远程 RISC-V 板子需要使用端口转发或访问板子 IP。
