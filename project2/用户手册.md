# Project2 用户手册

本文档面向项目使用者，说明如何编译、运行、查看结果，以及如何准备 UAC 和网页实时频谱功能。

## 1. 准备音频

将待分析音频放到仓库根目录的 `dataset` 文件夹。例如：

```text
dataset/1.wav
dataset/music.mp3
```

推荐先用 WAV 验证流程。若要处理 MP3、FLAC 等格式，请安装 FFmpeg。

## 2. 编译项目

Linux、macOS、RISC-V 平台：

```bash
cmake -S project2 -B project2/build -DCMAKE_BUILD_TYPE=Release
cmake --build project2/build
```

Windows：

```powershell
cmake -S project2 -B project2/build
cmake --build project2/build --config Release
```

## 3. 在 RISC-V 上启用 RVV

如果工具链支持 RVV，可以传入架构参数：

```bash
cmake -S project2 -B project2/build -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_CXX_FLAGS="-O3 -march=rv64gcv -mabi=lp64d"
cmake --build project2/build
```

运行时查看：

```text
RVV STFT acceleration: Enabled
```

如果显示 `Disabled`，说明当前编译器没有定义 `__riscv_vector`，需要检查 `-march`、ABI 和工具链版本。

## 4. 运行批量处理

Linux、macOS、RISC-V：

```bash
./project2/build/project2
```

Windows：

```powershell
.\project2\build\Release\project2.exe
```

程序会扫描 `DATASET_PATH` 指定目录，并将结果写入：

```text
project2/result/
```

## 5. 查看结果

假设输入文件是：

```text
dataset/1.wav
```

输出文件为：

```text
project2/result/1-spectrogram.png
project2/result/1-mel_spectrogram.png
project2/result/1-target_freq.svg
project2/result/1-target_freq.txt
```

PNG 和 SVG 可直接用系统图片查看器或浏览器打开。TXT 可用文本编辑器查看，每行包含：

```text
时间(秒)    幅度(dB)
```

## 6. 修改目标频率

编辑：

```text
project2/settings.h
```

修改：

```cpp
#define TARGET_FREQUENCY 1000.0f
```

然后重新编译并运行。

## 7. 连接 UAC 设备

当前项目已经定义实时输入抽象接口：

```cpp
class IAudioInputStream {
public:
    virtual bool start() = 0;
    virtual void stop() = 0;
    virtual int read(float* buffer, int frames) = 0;
};
```

启用步骤：

1. 安装 PortAudio 开发库。
2. 将 `settings.h` 中的 `ENABLE_UAC_INPUT` 改为 `1`。
3. 重新运行 CMake 配置和编译。
4. 将 USB Audio Class 设备接入系统。

当前主程序以文件批处理为主，实时输入接口用于后续把 UAC 数据接入 STFT 流水线。

## 8. 打开网页实时频谱

当前项目已经预留网页频谱推送接口。启用方法：

1. 将 `settings.h` 中的 `ENABLE_WEB_SPECTROGRAM` 改为 `1`。
2. 接入 WebSocket 后端库，例如 `libwebsockets`。
3. 重新编译运行。
4. 浏览器访问：

```text
http://localhost:9002
```

如果程序运行在远程 RISC-V 板子上，请使用端口转发或访问板子的实际 IP。

## 9. 常见问题

如果程序提示没有找到音频，请确认音频是否在 `dataset` 目录中。

如果 MP3 无法解码，请确认系统是否安装 `ffmpeg` 命令或 FFmpeg 开发库。

如果 RISC-V 编译 RVV 失败，请检查工具链是否支持 RVV 1.0，并确认 `-march=rv64gcv` 或对应参数是否正确。
